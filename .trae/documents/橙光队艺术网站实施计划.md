# 橙光队高端创意互动型网站 — 实施计划

## 摘要

为"橙光队"打造一个**极简高级感 + 特别艺术**的创意互动型静态网站。网站采用单页长滚动叙事结构，以"光的旅程"为核心视觉隐喻，融合 Three.js 粒子动画、GSAP 滚动驱动动画、视差滚动等高级效果，结合飞书中的真实素材和推送内容，展示橙光队的发展历程与团队文化。

**输出：** 纯静态 HTML/CSS/JS 文件，可直接部署到任意静态托管平台。

---

## 一、当前状态分析

### 数据来源
1. **飞书"橙光队"文件夹** — 包含文档、图片、PPT等素材（需通过 lark-drive/lark-doc/lark-wiki 技能读取）
2. **飞书电子表格** — `https://ecnoplhxkdub.feishu.cn/sheets/P7OusL5ZohJhI0tNBFkcu1AGnlx`（推送链接 + 活动记录）
3. **推送链接** — 表格中的URL，需爬取获取内容要点

### 技术约束
- 纯前端静态网站，无后端
- 需支持主流浏览器（Chrome/Firefox/Safari/Edge）
- 需响应式适配（桌面/平板/手机）
- 首屏加载 < 3秒

## 二、实施步骤

### 步骤 1：飞书数据采集

**目的：** 获取所有素材和内容数据

**操作：**
1. 使用 `lark-drive +search` 搜索"橙光队"文件夹，获取文件夹 token
2. 使用 `lark-drive` 列出文件夹内所有文件（文档、图片、PPT等）
3. 使用 `lark-sheets +info` 和 `lark-sheets +read` 读取电子表格内容，提取推送链接和活动记录
4. 使用 `lark-doc +fetch --api-version v2` 读取文档内容
5. 使用 `lark-drive +download` 下载图片和PPT素材到本地
6. 如有 Wiki 链接，使用 `lark-wiki +node-get` 解析后读取

**输出：** 本地素材库（图片、文档文本、表格数据、PPT文件）

### 步骤 2：推送链接爬取

**目的：** 从表格中的URL爬取推送内容要点

**操作：**
1. 从表格数据中提取所有推送链接URL
2. 使用 `web-content-fetcher` 或 `WebFetch` 逐个爬取推送页面
3. 对爬取失败的使用 `scrapling-official` 作为备选方案
4. 提取每个推送的：标题、日期、正文要点、关键信息
5. 按时间线排序，按主题分类

**输出：** 结构化的推送内容摘要列表

### 步骤 3：PPT素材提取

**目的：** 从飞书PPT中提取图文素材

**操作：**
1. 使用 `Powerpoint / PPTX` 技能读取下载的PPT文件
2. 提取关键页面的截图和文字内容
3. 使用 `ocr-local` 对图片型PPT页面进行OCR识别
4. 整理可用的图片素材和文案素材

**输出：** PPT中的图片素材 + 文字内容

### 步骤 4：素材整理与内容架构

**目的：** 将所有采集的数据整理为网站内容架构

**操作：**
1. 整理团队照片（活动照、合影、工作照）
2. 梳理时间线数据（活动日期、事件描述）
3. 归类推送内容（里程碑、团队活动、文化故事）
4. 提取关键成就和数据
5. 确定网站各板块的内容映射

**输出：** 完整的内容素材清单 + 网站内容映射表

### 步骤 5：高级网页效果研究

**目的：** 研究并确定网站将使用的高级视觉效果

**操作：**
1. 使用 `deep-researcher` 研究 Awwwards 获奖网站的设计趋势和高级效果
2. 使用 `agent-reach` 搜索艺术网站/博物馆网站的交互设计案例
3. 研究以下技术方案：
   - GSAP ScrollTrigger（滚动驱动动画）
   - Three.js 粒子系统（背景氛围）
   - Lenis 平滑滚动
   - 视差滚动效果
   - 文字揭示动画
   - 水平滚动画廊
4. 使用 `web-design-guidelines` 审查设计方案
5. 使用 `ui-ux-pro-max` 生成设计系统（色彩、字体、布局）

**输出：** 技术方案文档 + 设计系统规范

### 步骤 6：设计系统建立

**目的：** 建立完整的设计系统

**色彩方案（以"橙光"为灵感）：**
- 主色：`#FF6B35`（橙光）
- 背景：`#0A0A0A`（近黑）/ `#141414`（次级）
- 文字：`#F5F0EB`（暖白）/ `#8A8580`（暖灰）
- 点缀：`#C9A96E`（金色）/ `#F5E6D3`（奶油色）

**字体方案：**
- 标题：Noto Serif SC（思源宋体）— 优雅高级
- 正文：Noto Sans SC（思源黑体）— 清晰现代
- 诗意引言：Noto Serif SC — 衬线斜体

**动效规范：**
- 入场动画：0.8-1.2s，`power3.out`
- 悬停微交互：0.3-0.5s，`power2.inOut`
- 文字揭示：0.6-0.8s，逐字/逐行

**输出：** CSS变量定义 + 设计令牌

### 步骤 7：网站架构设计

**目的：** 确定页面结构和各板块设计

**页面结构（单页滚动叙事）：**

| 团队文化 | 照片墙 + 文化理念 | 瀑布流 + 悬停放大 |
| 活动画廊 | 精选照片 | 水平滚动 + Lightbox |
| 结尾 | 诗意收尾 + 联系方式 | 打字机效果 |

**导航：** 固定顶部毛玻璃导航 + 滚动进度条 + 移动端汉堡菜单

**输出：** HTML结构设计 + 各板块交互说明

### 步骤 9：前端开发 — JavaScript 交互与动画

**目的：** 实现所有高级交互效果

**核心动画模块：**

1. **`particles.js`** — Three.js 粒子系统
   - 1000-2000个橙色调粒子
   - Perlin 噪声驱动的有机流动
   - 鼠标交互响应
   - 滚动联动（改变密度/颜色/速度）

2. **`animations.js`** — GSAP 动画编排
   - 文字逐字揭示（SplitText 逻辑）
   - 视差滚动效果
   - 时间线 SVG 路径绘制
   - 数字计数动画
   - 水平滚动画廊（pin + scrub）
   - 元素入场动画（stagger）

3. **`navigation.js`** — 导航系统
   - 滚动方向判断（显示/隐藏导航）
   - IntersectionObserver 板块高亮
   - 平滑锚点滚动
   - 滚动进度条

4. **`main.js`** — 初始化协调
   - Lenis + GSAP ScrollTrigger 集成
   - 模块初始化顺序
   - 全局状态管理
   - resize 防抖处理

**输出：** 完整的 JS 交互代码

## 三、假设与决策

1. **飞书API已授权** — 用户已授权飞书访问，可直接使用 lark-cli 读取数据
2. **纯静态方案** — 不使用任何后端框架，所有效果通过前端技术实现
3. **CDN依赖** — GSAP、Three.js、Lenis 等通过 CDN 引入，无需构建工具
4. **单页设计** — 采用单页长滚动而非多页路由，更适合叙事式展示
5. **移动端降级** — Three.js 粒子效果在移动端使用 CSS 动画降级方案
6. **图片优化** — 所有图片转换为 WebP 格式，提供多分辨率 srcset
