# 橙光队首页艺术化重设计划

## 任务重述

将橙光队网站首页从"官方宣传页"转变为"艺术化叙事体验"，核心围绕"光"的概念：
- **核心隐喻**：一点点橙光粒子汇聚，最终形成橙光队
- **体验目标**：渐进式 reveal，避免开门见山的官方感
- **参考风格**：pudding.cool 的数据叙事、滚动驱动动画、粒子效果

## 借鉴 pudding.cool 的设计模式

### 1. 渐进式开场叙事
- **Scrollytelling 结构**：使用 IntersectionObserver 实现滚动触发的内容展示
- **Sticky 背景层**：背景可视化元素固定在视口内，随滚动变化
- **分步揭示**：内容按滚动位置逐步展示，而非一次性呈现

### 2. 粒子/汇聚效果实现方案
- **Canvas 粒子系统**：使用 HTML5 Canvas 实现光点粒子
- **汇聚动画**：粒子从随机位置向中心汇聚，形成"橙光队"字样或队徽
- **鼠标交互**：粒子对鼠标移动产生响应（排斥或吸引）

### 3. 性能优化实践
- IntersectionObserver 替代滚动事件监听
- `transform` 和 `opacity` 实现 GPU 加速动画
- 支持 `prefers-reduced-motion` 媒体查询

## 拟议变更

### 变更 1：重构 Hero 开场体验

**文件**: `index.html` (Hero Section ~1706行起)

**当前问题**:
- 直接显示"橙光队"标题，过于直白
- 缺乏"光汇聚"的概念表达

**改造方案**:
```
新 Hero 结构:
├── 阶段 1: 黑暗中的微光 (0-30% 滚动)
│   └── 零星橙光粒子在黑暗中漂浮
├── 阶段 2: 光的汇聚 (30-60% 滚动)
│   └── 粒子开始向中心聚集，形成光团
├── 阶段 3: 橙光初现 (60-80% 滚动)
│   └── 光团凝聚成"橙光"二字，发光效果
└── 阶段 4: 完整 reveal (80-100% 滚动)
    └── "橙光队"完整标题显现，副标题渐入
```

**技术实现**:
- 使用 Canvas 2D 实现粒子系统
- 粒子类包含：位置、速度、目标位置、大小、透明度
- 根据滚动进度控制粒子行为阶段
- 使用 GSAP ScrollTrigger 或 IntersectionObserver 驱动

### 变更 3：粒子系统实现

**文件**: `index.html` (JS 部分)

**核心类设计**:
```javascript
class LightParticle {
  constructor(canvas) {
    this.x = Math.random() * canvas.width;
    this.y = Math.random() * canvas.height;
    this.targetX = 0; // 汇聚目标
    this.targetY = 0;
    this.size = Math.random() * 2 + 1;
    this.opacity = Math.random() * 0.5 + 0.3;
    this.speed = Math.random() * 0.5 + 0.2;
  }
  
  update(scrollProgress) {
    // 根据滚动进度切换行为
    // 0-0.3: 漂浮
    // 0.3-0.6: 向中心汇聚
    // 0.6-1.0: 形成文字
  }
  
  draw(ctx) {
    ctx.beginPath();
    ctx.arc(this.x, this.y, this.size, 0, Math.PI * 2);
    ctx.fillStyle = `rgba(255, 107, 53, ${this.opacity})`;
    ctx.fill();
  }
}
```

## 技术决策

### 决策 1：粒子系统技术选型
**选项 A**: 纯 Canvas 2D API
- 优点：轻量、性能好、无需额外库
- 缺点：复杂效果实现较繁琐

**选项 B**: Three.js (3D)
- 优点：可实现更炫酷的3D效果
- 缺点：过重，与当前简约风格不符

**决策**: 使用 **Canvas 2D API**，足够实现所需的粒子效果，且与现有代码风格一致。

### 决策 2：滚动驱动方案
**选项 A**: GSAP ScrollTrigger (已引入)
- 优点：API 成熟，与现有 GSAP 动画一致
- 缺点：增加依赖

**选项 B**: IntersectionObserver (原生)
- 优点：原生支持，无额外依赖
- 缺点：实现滚动进度映射较复杂

**决策**: 使用 **GSAP ScrollTrigger**，项目已引入 GSAP，保持一致性。

### 决策 3：是否保留现有 Hero 内容
**决策**: 保留但重构展示方式
- 保留"橙光队"标题、副标题、slogan
- 改为渐进式 reveal，而非直接显示
- 保留 Canvas 背景层，替换为粒子系统
