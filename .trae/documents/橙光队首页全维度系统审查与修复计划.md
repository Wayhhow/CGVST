# 橙光队首页全维度系统审查与修复计划

## Summary

对橙光队首页 `index.html`（约 3873 行，含 4 个 Canvas 层、GSAP + Lenis + 自定义光标 + 粒子系统）进行**全维度系统审查**，覆盖**内容准确性、视觉设计、代码质量、性能与无障碍**四大类问题。计划分为 5 个阶段（探查→确认→修复→验证→交付），任何**展示内容（数字/日期/名称/链接/描述）的修改必须先与用户确认**（用户最熟悉内容），其他代码/CSS/性能/无障碍类问题可直接修复。最后用**网络抓取、截图仿真、单元测试**三层验证确保零瑕疵。

## Assumptions & Decisions

### 假设
1. **用户是内容权威**：所有数字/日期/名称/链接/描述的最终判定权属用户，AI 不得擅自改写
2. **部署地址**：未确认网站部署在哪个域名（GitHub Pages? 自建服务器？）—— 部分验证项（og:image 绝对地址）需先问清
3. **CDN 允许 SRI**：假设当前 CSP 允许外链脚本（若启用严格 CSP，则 SRI 修复是必须的）
4. **性能基准**：目标设备是 2020 年后中端笔电 + iPhone 12 及以上
5. **降级策略**：低端设备不需要 Canvas 优化（如 `will-change` 数量限制），按现代浏览器默认性能走

### 决策
1. **不引入新框架**：保持纯 HTML/CSS/JS + GSAP + Lenis 栈
2. **不拆分文件**：用户偏好单文件部署（从历史 plan 看 index.html 内联一切）
3. **不重写视觉**：只在原设计基础上修复问题，不重做品牌
4. **测试工具栈**：Node + jsdom + Playwright（headless），不引入 Vitest/Jest（项目无 npm init）
5. **OCR 不纳入**：用户没在多选里勾选 OCR（仅选 网络抓取/截图/单元测试），故不调用 ocr-local

### 待用户确认的开放问题
- 网站部署 URL（用于 og:image 绝对地址）
- 是否允许给 CDN 加 SRI
- "More Honors" 卡片是否改用 emoji 图标
- Preloader 副标题是否中文化
